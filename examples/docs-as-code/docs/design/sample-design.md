# 設計: docs/ ポータル（index.md）の導入

## 目的

- 文書が増えても「探せる状態」を維持できるようにする。

## 背景

- README と個別ドキュメントが増えると、入口が増えて迷いやすい。

## スコープ

- 含めること：`docs/index.md` を入口として整備し、主要文書へリンクする。
- 含めないこと：検索基盤（外部ツール）の導入。

## 要求（Requirements）

- 主要カテゴリ（ADR/議事録/設計/Runbook）に 1 クリックで到達できる。
- README から docs/index.md へリンクできる。

## 仕様概要

- docs/index.md に目次（リンク）と最小運用ルールを置く。

## 方式（Approach）

- `docs/index.md` は手で編集する（自動生成は当面しない）。
- カテゴリ別ディレクトリを固定する（`adr/`, `meeting-notes/` など）。

## セキュリティ・権限

- 公開/非公開の境界は別途 ADR で決める（この設計では扱わない）。

## 運用

- 文書の追加時は docs/index.md のリンクも更新する。

## テスト

- 主要リンクが全て辿れる（リンク切れがない）ことを確認する。

## リリース計画

- Pull Request で導入し、マージ後に README のリンクを追加する。
