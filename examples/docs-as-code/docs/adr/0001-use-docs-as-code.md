# ADR-0001: 文書は docs/ 配下で管理し、PRでレビューする

## ステータス

- 承認済

## 日付

- 2025-01-21

## 背景（Context）

- 仕様・運用手順がチャットや個人メモに分散すると、最新版が分からず誤操作につながる。
- 文書を更新するたびに「誰が確認するか」が曖昧で、品質が維持しづらい。

## 決定（Decision）

- 文書は `docs/` 配下で管理し、変更は Pull Request でレビューする。

## 理由（Rationale）

- 変更理由（Issue）と変更内容（PR）が紐づき、根拠を追いやすい。
- レビューの観点が揃い、属人化しにくい。

## 代替案（Alternatives）

- Wiki のみで運用する：編集はしやすいが、レビュー・履歴・運用ルールが曖昧になりやすい。
- 共有ドライブで管理する：閲覧性は高いが、変更差分やレビューの仕組みが不足する。

## 影響（Consequences）

- Pull Request の運用が前提になるため、最小限のルール（テンプレ・レビュー担当）を整備する必要がある。
